# Contributing

Thanks for improving Obi-Harness.

Keep contributions minimal, practical, and grounded in real usage.

## Contribution standard

This repo is not trying to become a large framework.

Good contributions usually do one of these:

- fix a factual inconsistency
- improve clarity in the template or docs
- add a protocol that has already been validated in real use
- improve examples so they are more honest and more useful
- simplify something that has become confusing

Please avoid:

- adding complexity without evidence
- introducing files that do not clearly earn their place
- adding speculative abstractions
- turning the repo into a branded marketing artifact

## Preferred contribution flow

1. Open an issue or PR that explains the real problem.
2. Show the concrete confusion, failure mode, or friction.
3. Keep the fix as small as possible.
4. Update examples or docs when behavior changes.

## Guiding principles

- Start small.
- Stable core, evolving edge.
- Failures are first-class.
- Observe, propose, review, adopt.
- Real patterns beat theoretical elegance.
