# Benchmarks

## Quality benchmarks

- Every user-facing action has loading and error states
- No hardcoded customer-facing strings
- Role boundaries are enforced consistently
- Feature work does not break adjacent flows

## Verification benchmarks

- changed flow works end to end
- failure states are tested
- access rules are verified for each user role

## What done means

- feature merged
- QA passed
- blockers cleared
- status updated
