# Examples

These examples are miniature workspaces, not full production projects.

Their job is to show how Obi-Harness feels in practice:

- what the day-one files look like
- how a website project differs from a product project
- how status, decisions, and agent rules become concrete

They are intentionally small, but they should still be internally honest.
If an example references a file, that file should exist in the example.

## Included examples

### Website project

Shows a public-site workspace with:

- brand and design constraints
- page-quality benchmarks
- active workstream tracking
- website-oriented decisions

### Product project

Shows an internal product workspace with:

- product and API constraints
- feature and bug benchmarks
- active workstream tracking
- architecture and UX decisions
